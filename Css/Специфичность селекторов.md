Специфичность (Specificity) в CSS — это алгоритм, с помощью которого браузер определяет, какое именно правило применить к элементу, если к нему подходят сразу несколько разных CSS-селекторов.

Простыми словами: это «вес» или «сила» селектора. Кто сильнее, того стили и побеждают.

---

## Как рассчитывается вес селектора?

Специфичность можно представить в виде четырехзначного счетчика из четырех колонок: `[ 0, 0, 0, 0 ]`. Чем левее находится цифра, тем она сильнее. Колонки никогда не смешиваются (10 классов никогда не перевесят 1 идентификатор).

Вот за что отвечает каждая колонка (от самой сильной к самой слабой):

1. Inline-стили `[ 1, 0, 0, 0 ]` — стили, написанные прямо в HTML-теге через атрибут `style` (например, `<div style="color: red;">`).
2. ID (Идентификаторы) `[ 0, 1, 0, 0 ]` — селекторы, начинающиеся с решетки (например, `#header`).
3. Классы, Атрибуты и Псевдоклассы `[ 0, 0, 1, 0 ]` — селекторы классов (`.menu`), селекторы атрибутов (`[type="text"]`) и псевдоклассы (`:hover`, `:focus`).
4. Теги и Псевдоэлементы `[ 0, 0, 0, 1 ]` — обычные имена HTML-тегов (`div`, `p`, `h1`) и псевдоэлементы (`::before`, `::after`).

_Примечание: Универсальный селектор (`*`), комбинаторы (`+`, `>`, `~`) и псевдокласс `:where()` имеют нулевой вес `[ 0, 0, 0, 0 ]`._

---

## Примеры для сканирования и сравнения:

- `div` ➔ `[ 0, 0, 0, 1 ]` (один тег)
- `div .card` ➔ `[ 0, 0, 1, 1 ]` (один класс + один тег)
- `.sidebar .menu a:hover` ➔ `[ 0, 0, 3, 1 ]` (два класса + один псевдокласс + один тег)
- `#main-nav a` ➔ `[ 0, 1, 0, 1 ]` (один ID + один тег) — Победит предыдущий вариант, так как ID стоит в более сильной колонке.

## Исключения и супероружие

## 1. Псевдоклассы `:not()`, `:has()` и `:is()`

Сами эти псевдоклассы веса не имеют, но они принимают вес самого тяжелого селектора, переданного внутрь их круглых скобок.

- `:not(.active)` ➔ имеет вес одного класса `[ 0, 0, 1, 0 ]`.
- `:not(#page-title)` ➔ имеет вес одного ID `[ 0, 1, 0, 0 ]`.

## 2. Флаг `!important`

Это не селектор, а «ядерная кнопка» для конкретного CSS-свойства. Он ломает стандартный подсчет специфичности и заставляет браузер выбрать именно это свойство, несмотря ни на какие ID и inline-стили. Перебить `!important` может только другой `!important`, у которого селектор окажется специфичнее.

## 3. Что если вес абсолютно одинаковый?

Если два селектора имеют абсолютно равный вес (например, оба `.dark-theme p`), побеждает тот, который написан в CSS-файле ниже (позже).

**Пример расчёта специфичности:**

Фраза «подходят несколько селекторов» означает, что в твоем CSS-файле написано несколько разных правил, но все они нацелены на один и тот же конкретный HTML-элемент на странице. Браузер видит их все, но у него возникает конфликт: какое правило применить, если они противоречат друг другу?

Давай посмотрим на живом примере.

## HTML-структура:

У нас есть всего одна обычная кнопка в меню:

```html
<nav id="main-nav">
  <button class="btn active">Купить</button>
</nav>
```

## CSS-стили:

Ты (или разные разработчики в команде) написали в стилях следующие правила для цвета этой кнопки:

```css
/* Правило №1: Выбираем по тегу */
button {
  background-color: gray;
}

/* Правило №2: Выбираем по классу */
.btn {
  background-color: blue;
}

/* Правило №3: Выбираем по двум классам сразу */
.btn.active {
  background-color: green;
}

/* Правило №4: Выбираем через ID родителя */
#main-nav button {
  background-color: red;
}
```

## Что делает браузер в этот момент?

Когда браузер рендерит эту кнопку, он «понимает», что все 4 правила подходят к ней:

1. Это тег `button`? Да (Правило 1 подходит).
2. У неё есть класс `.btn`? Да (Правило 2 подходит).
3. У неё есть одновременно классы `.btn` и `.active`? Да (Правило 3 подходит).
4. Находится ли она внутри элемента с `id="main-nav"`? Да (Правило 4 подходит).

Все 4 правила требуют покрасить фон в разный цвет: серый, синий, зеленый или красный. Кнопка не может быть разноцветной одновременно.

## Как специфичность решает этот конфликт?

Браузер включает счетчик специфичности `[ID, Класс, Тег]` для каждого подошедшего селектора и взвешивает их:

- Правило 1 (`button`) ➔ `[0, 0, 1]` _(1 тег)_ — Самый слабый.
- Правило 2 (`.btn`) ➔ `[0, 1, 0]` _(1 класс)_ — Сильнее, чем тег.
- Правило 3 (`.btn.active`) ➔ `[0, 2, 0]` _(2 класса)_ — Сильнее, чем один класс.
- Правило 4 (`#main-nav button`) ➔ `[1, 0, 1]` _(1 ID + 1 тег)_ — Абсолютный победитель!

Поскольку у Правила №4 в колонке ID стоит единица `[1, 0, 1]`, оно мгновенно побеждает все остальные правила, даже если в них будет написано 20 классов подряд (например, `.btn.active.large.theme-dark...` всё равно проиграет одному ID).

В итоге кнопка покрасится в красный цвет. Стили остальных правил браузер просто зачеркнет в панели разработчика (DevTools).
