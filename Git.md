**Git** - распределенная система управления версиями (VCS).

Распределенная система - каждый разработчик имеет полную копию репозитория (истории проекта) на своем компьютере.

**Git Index** - список файлов и изменений, подготовленных для следующего коммита.

**Коммит**  — это «снимок» (snapshot) состояния всех файлов проекта в репозитории на определенный момент времени.

Отличие `git fetch` от `git pull`

git fetch и git pull служат для получения изменений из удаленного репозитория, но делают это по-разному. Fetch только **скачивает** новые данные, не меняя рабочую копию, что безопасно. Pull **скачивает и сразу сливает** (merge) эти изменения в текущую ветку. 

**Основные отличия:**

- **Git Fetch:** Скачивает коммиты, файлы и ссылки из удаленного репозитория, обновляя состояние `origin/master`, но не затрагивает ваши локальные файлы. Позволяет увидеть, что сделали другие, перед тем как объединить изменения.
- **Git Pull:** Это, по сути, комбинация `git fetch` + `git merge`. Она автоматически обновляет локальную ветку.
