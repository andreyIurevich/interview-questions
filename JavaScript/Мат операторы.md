В JavaScript оператор `+` — это «двуликий» оператор. Он выполняет две совершенно разные математические операции:

1. Сложение (если оба операнда — числа).
2. Конкатенацию/склеивание (если хотя бы один из операндов — строка).

## Главное правило движка JS: Строка всегда побеждает

Разработчики языка заложили в алгоритм оператора `+` простое правило: если примитивы нельзя сложить как числа, приоритет отдается строкам.

Когда движок JavaScript видит выражение `1 + "1"`, он выполняет следующий алгоритм (согласно спецификации):

1. Он смотрит на левую часть: `1` (тип Number).
2. Он смотрит на правую часть: `"1"` (тип String).
3. Движок видит, что один из участников — строка. В этот момент срабатывает правило: _«Если хотя бы один операнд является строкой, переключить режим оператора в "склеивание строк", а второй операнд принудительно привести к строке»_.
4. Число `1` превращается в строку `"1"`.
5. Происходит склеивание: `"1" + "1"` ➡️ `"11"`.

## Почему JS не пытается сначала распарсить строку в число?

Потому что это было бы небезопасно и непредсказуемо.

Представь, если бы JS всегда пытался превратить строку в число. Что бы тогда произошло в выражении `1 + "привет"`?

- Строка `"привет"` не может стать числом. Она бы превратилась в специальное значение `NaN` (Not a Number).
- Выражение превратилось бы в `1 + NaN`.
- Любая математическая операция с `NaN` возвращает `NaN`.
- В итоге `1 + "привет"` возвращало бы `NaN`.

Чтобы разработчики не получали `NaN` на каждый чих при случайном сложении текста и чисел, создатели языка решили: безопаснее и логичнее просто склеить их как текст. Текст `"1привет"` гораздо понятнее для отладки, чем абстрактный `NaN`.

## А как ведут себя другие операторы? (Минус, Умножение, Деление)

А вот со всеми остальными математическими операторами (`-`, `*`, `/`) логика работает ровно наоборот — там всегда побеждают числа!

У операторов минус, умножить или разделить нет второй роли (они не умеют склеивать текст). Их единственная задача — чистая математика. Поэтому они всегда принудительно превращают всё в числа:

```javascript
"5" - 2  // ➡️ 3  (Строка "5" успешно превратилась в число 5)
"5" * "2" // ➡️ 10 (Обе строки превратились в числа)
"5" - "привет" // ➡️ NaN (Текст "привет" не смог стать числом, получили ошибку)
```
