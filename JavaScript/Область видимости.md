Область видимости определяет, где в коде программы будут доступны переменные и функции. В JavaScript есть два типа области видимости — глобальная и локальная.

Создаются области видимости во время выполнения программы. Самая первая область, которая создаётся и которая включает в себя все остальные называется глобальной.

Если переменная создаётся внутри объявления функции, то её область видимости определяется как локальная и ограничивается этой функцией.

##### Задачи

```js
var o = {};
var i = 0;

(function () {
	o.a = 'a';
	i++;
})();

console.log(o, i); // { a: 'a' }, 1

(function (o, i) {
	o.b = 'b';
	i++;
	console.log('i = ', i); // i = 2
})(o, i);

console.log(o, i); // { a: 'a', b: 'b' }, 1
```


При вызове функции fnA() у нас создаётся локальная область видимости, которая будет иметь в качестве ссылки глобальную область. Почему так? **Потому что внешняя область определяется в зависимости от того, где объявлена функция, а не вызвана**. А так как функция объявлена в глобальной области видимости, то не зависимого того где она вызвана, она будет содержать в качестве ссылки - ссылку на внешнюю область видимости, в зависимости от того где она объявлена.

```js
let num = 10;

function fnA() {
  console.log(num);
}

function fnB() {
  let num = 20;
  fnA();
}
fnB(); 

// Вывод num = 10
```


```js
{
  var a = 5;
  let b = 7;
  function myFunc() {
    var c = 5;
  }
}

myFunc();
console.log(a); // 5
console.log(b); // Uncaught ReferenceError: b is not defined
console.log(c); // Uncaught ReferenceError: c is not defined
```

Ключевое слово var в отличие от let и const, создаёт глобальные переменные, которые будут являться свойствами глобального объекта window в веб-браузере и global в Node.js
Переменные, созданные в глобальной области видимости с помощью ключевых слов let, const и class, не являются свойствами глобального объекта.

```js
var a = 5;
let b = 7;

console.log(window.a); // 5
console.log(window.b); // undefined
```

Переменная myName в этом примере будет объявлена автоматически, причём это будет сделано в глобальной области видимости. Тем самым после завершения выполнения функции sayHello(), эта переменная будет нам доступна и мы можем вывести её значение в консоль.

```js
function sayHello() {
  myName = 'Вася';
  console.log(`Привет, ${myName}!`);
}

sayHello(); // Привет, Вася!
console.log(myName); // Вася
```

```js
var obj = { a: 1 };

function f2(o) {
	o = { hello: 1 };
}

f2(obj);

console.log(obj); // { a: 1 }
```

Локальная переменная `o` — это ключевой фактор. Но чтобы до конца понять этот механизм, нужно разобрать, как JavaScript работает с объектами и ссылками при передаче в функцию.

В JavaScript аргументы всегда передаются по значению. Но когда аргументом является объект, этим «значением» является адрес ссылки на объект в памяти.

Давай пошагово разберем, что происходит в памяти компьютера во время выполнения твоего кода:

## Шаг 1: Создание объекта в памяти

```javascript
var obj = { a: 1 };
```

- В куче (памяти) создается объект `{ a: 1 }`.
- Переменная `obj` сохраняет не сам объект, а ссылку (адрес) на него, например: `Ссылка_№123`.

## Шаг 2: Вызов функции и копирование ссылки

```javascript
f2(obj);
```

- Когда ты вызываешь `f2(obj)`, JavaScript берет значение из переменной `obj` (то есть адрес `Ссылка_№123`) и копирует его в локальную переменную-аргумент `o`.
- В этот момент внутри функции переменная `o` указывает на тот же самый объект в памяти (`Ссылка_№123`). Если бы ты внутри функции написал `o.a = 5`, то оригинальный объект изменился бы.

## Шаг 3: Перезапись локальной ссылки (Тот самый момент!)

```javascript
function f2(o) {
  o = { hello: 1 }; // Перезапись!
}
```

- Строчка `o = { hello: 1 }` создает в памяти абсолютно новый объект (например, `Ссылка_№999`).
- Знак равенства `=` берет эту новую ссылку и записывает её в локальную переменную `o`.
- Что произошло с оригиналом? Ничего! Мы просто оборвали связь локальной переменной `o` со старым объектом (`Ссылка_№123`) и направили её на новый. Оригинальная переменная `obj` снаружи функции как указывала на `Ссылка_№123`, так и продолжает указывать.

## Шаг 4: Вывод в консоль

```javascript
console.log(obj); // { a: 1 }
```

Функция завершилась, её локальная переменная `o` уничтожилась. Внешний объект `obj` остался в своем первозданном виде.
