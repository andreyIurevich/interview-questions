Главная концепция: В JavaScript нет классического классового наследования (как в Java или C++). Классы в JS — это «синтаксический сахар» над прототипным наследованием. Объекты связываются друг с другом напрямую через ссылки в цепочки.

## 🧱 1. Ключевые термины (Не путай их на собеседовании!)

- `[[Prototype]]` — скрытое, внутреннее системное свойство объекта. Это физическая ссылка на объект-родитель.
- `__proto__` — исторический геттер/сеттер для доступа к `[[Prototype]]`. Сегодня его использовать в коде не рекомендуется (плохо для производительности).
- `prototype` — свойство, которое есть только у функций-конструкторов (и обычных функций, объявленных через `function`). Оно указывает, какой `[[Prototype]]` получат объекты, созданные через `new ИмяФункции()`. Обычные объекты свойства `prototype` не имеют.

---

## ⛓️ 2. Как работает цепочка прототипов (Prototype Chain)

Когда мы пытаемся прочитать свойство `obj.prop`:

1. JS ищет `prop` внутри самого объекта `obj`.
2. Если не находит, он делает шаг по ссылке `[[Prototype]]` к родителю.
3. Повторяет поиск. Если доходит до конца (`Object.prototype`) и ничего не находит, возвращает `undefined`.

> Важно: Цепочка прототипов всегда заканчивается на `Object.prototype`, а прототипом самого `Object.prototype` является `null` (`Object.prototype.__proto__ === null`). Это тупик, конечная точка.

## ⚙️ 3. Три способа создания объектов с прототипом

```javascript
const animal = { eats: true };

// Способ А: Современный и правильный (ES6+)
const rabbit = Object.create(animal); 

// Способ Б: Старый (через функции-конструкторы)
function Rabbit() {}
Rabbit.prototype = animal; // Привязка
const whiteRabbit = new Rabbit();

// Способ В: Динамическое изменение (Плохая практика, тормозит движок V8)
const owl = {};
Object.setPrototypeOf(owl, animal); 
```

## 🔍 4. Поиск свойств: Собственные vs Прототипные

- Метод `obj.hasOwnProperty('key')` — проверяет наличие свойства только в самом объекте, игнорируя прототипы.
- Оператор `'key' in obj` — проверяет наличие свойства и в самом объекте, и во всей цепочке его прототипов.

```javascript
console.log('jumps' in rabbit);        // true (есть в самом rabbit)
console.log('eats' in rabbit);         // true (нашел в прототипе animal)
console.log(rabbit.hasOwnProperty('eats')); // false (свойство не родное, а унаследованное)
```

## 🔄 5. Особенности цикла `for...in`

- Главная ловушка: Цикл `for...in` перебирает как собственные, так и унаследованные свойства объекта из цепочки прототипов.
- Почему он не выводит стандартные методы вроде `toString`? Потому что встроенные методы имеют скрытый флаг дескриптора `enumerable: false` (неперечисляемые). Цикл `for...in` видит только перечисляемые свойства.

Как правильно обходить только свои свойства на собеседовании:

```javascript
// Вариант 1: Фильтрация в цикле
for (let key in rabbit) {
  if (rabbit.hasOwnProperty(key)) {
    console.log(key); // Только собственные
  }
}

// Вариант 2: Использование Object.keys() (Игнорирует прототипы по умолчанию)
Object.keys(rabbit).forEach(key => console.log(key));
```

## 👑 6. Вопросы «на засыпку» от интервьюера

1. Что будет, если перезаписать унаследованное свойство? (`rabbit.eats = false`)
    
    - _Ответ:_ Прототип `animal` не изменится. Изменение/запись всегда происходят в самом объекте. В `rabbit` создастся собственное свойство `eats: false`, которое просто «перекроет» (затеняет / shadowing) свойство из прототипа.
    
2. Где берут свой `this` методы из прототипа?
    
    - _Ответ:_ Если мы вызовем унаследованный метод `rabbit.walk()`, то `this` внутри метода `walk` будет ссылаться на `rabbit` (на объект перед точкой), а не на `animal`, где этот метод физически объявлен.
    
3. Что такое `Object.create(null)`?
    
    - _Ответ:_ Это создание абсолютно чистого объекта, у которого нет прототипа вообще (даже `Object.prototype`). У такого объекта не будет методов `toString()`, `hasOwnProperty()` и т.д. Его часто используют как чистые «словари» для хранения данных, чтобы избежать конфликтов с системными именами.
    

## 🎛️ 7. Дескрипторы свойств (Скрытые настройки ключей)

В JavaScript каждое свойство объекта — это не просто пара `ключ: значение`. У каждого ключа есть скрытый объект-конфигуратор, который называется дескриптором. Он содержит 4 основных флага:

1. `value` — само значение свойства (по умолчанию `undefined`).
2. `writable` — можно ли изменять значение (`true` / `false`).
3. `enumerable` — будет ли свойство видно в циклах `for...in` и методе `Object.keys()` (`true` / `false`).
4. `configurable` — можно ли удалить это свойство из объекта, а также можно ли изменять сам этот дескриптор в будущем (`true` / `false`).

> Важнейшее правило: Если ты создаешь свойство обычным путем (`obj.a = 1`), все три флага (`writable`, `enumerable`, `configurable`) автоматически устанавливаются в `true`.  
> Но если ты создаешь свойство через специальный метод `Object.defineProperty`, все неуказанные флаги по умолчанию станут `false`!

## 🛠️ Как управлять дескрипторами в коде:

```javascript
const user = { name: 'Иван' };

// 1. Как ПОСМОТРЕТЬ дескриптор свойства:
console.log(Object.getOwnPropertyDescriptor(user, 'name'));
/* Выведет:
{
  value: 'Иван',
  writable: true,
  enumerable: true,
  configurable: true
}
*/

// 2. Как СОЗДАТЬ или ИЗМЕНИТЬ свойство с жесткими настройками:
Object.defineProperty(user, 'id', {
  value: 777,
  writable: false,      // Запрещаем перезаписывать ID
  enumerable: false,    // Скрываем из циклов for...in и Object.keys()
  configurable: false   // Запрещаем удалять (delete user.id) и менять этот дескриптор
});

user.id = 999;          // Игнорируется (в use strict вызовет ошибку TypeError)
delete user.id;         // Игнорируется (возвращает false, ID не удалится)
console.log(user.id);   // Всё еще 777
```

## 🔒 8. Глобальное управление объектом (Заморозка)

В JavaScript есть три встроенных метода для быстрой настройки дескрипторов сразу у всего объекта:

1. `Object.preventExtensions(obj)`
    
    - _Что делает:_ Запрещает добавлять новые свойства в объект.
    - _Старые свойства:_ Можно изменять и можно удалять.
    
2. `Object.seal(obj)` (Запечатать)
    
    - _Что делает:_ Запрещает добавлять новые свойства и удалять старые.
    - _Под капотом:_ Ставит всем существующим свойствам флаг `configurable: false`.
    - _Старые свойства:_ Всё еще можно изменять (если у них был `writable: true`).
    
3. `Object.freeze(obj)` (Заморозить) — _Самый популярный метод для иммутабельности_
    
    - _Что делает:_ Делает объект абсолютно неизменяемым. Нельзя добавить, нельзя удалить, нельзя поменять значение.
    - _Под капотом:_ Ставит всем свойствам `configurable: false` и `writable: false`.
    

⚠️ Ловушка «Заморозки» (На засыпку):  
Методы `freeze`, `seal` и `preventExtensions` являются поверхностными (shallow). Если внутри объекта есть вложенный объект, он _не_ заморозится!

```javascript
const config = Object.freeze({
  theme: 'dark',
  api: { url: 'https://api.com' } // Вложенный объект
});

config.theme = 'light';    // Не сработает (свойство заморожено)
config.api.url = 'evil.com'; // СРАБОТАЕТ! Вложенный объект остался изменяемым.
```

_Решение:_ Чтобы заморозить объект полностью, нужно писать рекурсивную функцию «глубокой заморозки» (Deep Freeze).
