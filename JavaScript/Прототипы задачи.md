Оператор `new` устроен так:

1. Создаёт новый объект.
2. Устанавливает ему `[[Prototype]] = Constructor.prototype`.
3. Вызывает конструктор.
4. Если конструктор вернул объект, именно он и станет результатом `new`. Если конструктор возвращает примитив, то оператор `new` **игнорирует** это значение.

```js
function User(name) {
  this.name = name;

  return {
    name: "Admin",
  };
}

const user = new User("Alex");

console.log(user.name); // "Alex"
console.log(user instanceof User); // false
```

```js
function User(name) {
  this.name = name;

  return 100;
}

const user = new User("Alex");

console.log(user.name); // "Alex"
console.log(user instanceof User); // true
```

**Правило, которое стоит запомнить**

Если свойство найдено в прототипе:

- **Чтение** (`obj.x`) — идёт по цепочке прототипов.
- **Запись** (`obj.x = ...`) — создаёт (или изменяет) **собственное** свойство объекта, а не свойство в прототипе (если нет сеттеров).
- **Изменение объекта**, найденного в прототипе (`obj.arr.push()`, `obj.map.set()`, `obj.obj.key = ...`) — изменяет сам объект, потому что это одна и та же ссылка.

```js
function Parent() {}

Parent.prototype.count = 0;

function Child() {}

Child.prototype = Object.create(Parent.prototype);
Child.prototype.constructor = Child;

const a = new Child();

a.count++;
```

Происходит примерно следующее:

```js
const temp = a.count; // чтение, нашли 0 в Parent.prototype
a.count = temp + 1;   // запись, создали собственное свойство count = 1
```
