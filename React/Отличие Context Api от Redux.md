Context API и Redux часто путают, потому что **оба решают проблему проброса данных**. Но по сути это **инструменты разного уровня**.

Context API — механизм передачи данных  
Redux — полноценное состояние приложения (state management)

**Использование**

**Context API** используется для:
1. темы (dark / light)
2. текущего пользователя
3. локали
4. feature flags
5. данных, которые редко меняются

**Redux** используется для:
1. глобального бизнес-состояния
2. сложной логики
3. кэширования данных
4. синхронизации между экранами
5. предсказуемых изменений state

**Рендер**

Context Api: Любое изменение value → ререндер всех consumers
Redux: Компонент подписывается только на нужный срез state (через useSelector)

**Архитектура**

Redux: чёткая архитектура (middleware и т.д.)
Context Api: Нет встроенной архитектуры

**Отладка**

Redux: redux-devtools
Context Api: сложно отслеживать изменения
