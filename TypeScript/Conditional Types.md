
**Conditional Types (Условные типы)** — это аналог тернарного оператора `if/else` (`условие ? если_true : если_false`), но работающий исключительно на уровне типов.

Они позволяют TypeScript принимать решения и динамически менять финальный тип в зависимости от других типов.

## Синтаксис

```typescript
T extends U ? X : Y
```

- `T extends U` — проверка (условие): «Можно ли тип `T` присвоить типу `U`?» (является ли `T` подтипом `U`).
- `X` — тип, который вернется, если условие выполняется (`true`).
- `Y` — тип, который вернется, если условие не выполняется (`false`).

## Простой пример из реальной жизни

Представь функцию, которая принимает на вход либо ID пользователя (число), либо имя пользователя (строку). Нам нужно, чтобы тип ответа автоматически подстраивался под входные данные: если передали строку — возвращаем объект пользователя, если число — возвращаем только статус.

```typescript
// Создаем условный тип-функцию
type IsString<T> = T extends string ? "Это строка" : "Это НЕ строка";

type Result1 = IsString<string>; // Результат: "Это строка"
type Result2 = IsString<number>; // Результат: "Это НЕ строка"
```

## Условные типы + Ключевое слово `infer`

Самая большая сила условных типов раскрывается вместе со словом `infer` (от английского _inference_ — вывод, угадывание). Оно позволяет попросить TypeScript: _«Если тип совпал, загляни внутрь него, угадай, что там за скрытый тип, и запиши его в новую переменную»_.

Давай напишем встроенный в TypeScript тип `Awaited<T>`, который достает чистый тип данных из Промиса:

```typescript
// Читается так: если T — это Promise, то "узнай" (infer) тип данных внутри него (U) 
// и верни этот U. Иначе верни сам T.
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;

type Post = { id: number; title: string };

// Проверяем:
type AsyncData = Promise<Post>;
type CleanData = UnwrapPromise<AsyncData>; 
// Магия! Сработал infer, и тип CleanData стал равен строго: Post

type NormalData = number;
type NotAsyncData = UnwrapPromise<NormalData>; 
// Условие не выполнилось (number — это не Promise), вернулся сам T: number
```

## Дистрибутивность (Как условные типы работают с `Union` типами)

Если передать в условный тип объединение (Union типа `A | B | C`), TypeScript автоматически применит условие к каждому элементу отдельно.

Помнишь тип `never`, который мы обсуждали? С помощью условных типов и `never` можно сделать мощный фильтр (так под капотом устроен встроенный тип `Exclude<T, U>`):

```typescript
// Удаляем тип U из типа T
type RemoveType<T, U> = T extends U ? never : T;

type Cleaned = RemoveType<string | number | boolean, boolean>;
// Под капотом TypeScript проверяет каждый тип отдельно:
// 1. string extends boolean? Нет -> string
// 2. number extends boolean? Нет -> number
// 3. boolean extends boolean? Да -> never (исчезает)
//
// Итог: string | number
```

**Реализация ИЛИ в условных типах**

Прямого оператора «ИЛИ» (как `||` в JavaScript) внутри синтаксиса условных типов TypeScript не существует.

Однако логику «ИЛИ» можно легко реализовать тремя разными способами, в зависимости от твоей задачи.

---

## Способ 1. Через объединение (Union types) — Самый частый вариант

В TypeScript проверка `A extends B` означает «является ли `A` подтипом `B`». Если в праве части (`B`) указать Union-тип через вертикальную черту `|`, компилятор проверит, подходит ли `A` хотя бы под один из этих типов. Это и есть чистое «ИЛИ».

```typescript
// Проверяем: T — это строка ИЛИ число?
type IsStringOrNumber<T> = T extends string | number ? "Да" : "Нет";

type Test1 = IsStringOrNumber<string>; // "Да"
type Test2 = IsStringOrNumber<number>; // "Да"
type Test3 = IsStringOrNumber<boolean>; // "Нет"
```

---

## Способ 2. Вложенные тернарные операторы (Если первое ИЛИ если второе)

Если тебе нужно не просто проверить тип, а вернуть разные значения для каждого условия, или если условия проверяют разные вещи, то используют вложенные (`else if`) проверки.

```typescript
// Если строка — возвращаем один тип, если число — другой, иначе — third
type Process<T> = T extends string 
  ? "Строковое значение" 
  : T extends number 
    ? "Числовое значение" 
    : "Что-то другое";
```

---

## Способ 3. Проверка «ИЛИ» для нескольких разных дженериков

Если тебе нужно проверить, правда ли, что `T` равен строке ИЛИ `U` равен числу (проверка двух разных переменных), можно скомбинировать их в один кортеж:

```typescript
type CheckTwo<T, U> = [T, U] extends [string, any] | [any, number] 
  ? "Хотя бы одно условие совпало" 
  : "Ничего не совпало";
```

_Здесь мы проверяем пару `[T, U]`. Условие сработает, если либо первый элемент — строка, либо второй — число._
