Дженерики — это, пожалуй, самая мощная тема в TypeScript. Если обычные функции работают со значениями (принимают числа/строки и возвращают их), то дженерики — это «функции для типов» (они принимают типы как аргументы и возвращают новые типы).

---

## 📑 Шпаргалка: Generics (Дженерики) в TypeScript

Generic (Универсальный тип / Параметризованный тип) — это инструмент, который позволяет создавать компоненты (функции, интерфейсы, классы), способные работать с различными типами данных, сохраняя при этом строгую типизацию (в отличие от `any`).

> Главный ментальный сдвиг: Относись к букве `<T>` внутри треугольных скобок как к обычной переменной, в которую вместо значения (`42` или `"hello"`) записывается тип данных (`number` или `string`).

---

## 🛠 1. Базовый синтаксис (Дженерики в функциях)

Дженерики позволяют связать тип входящего аргумента с типом возвращаемого значения.

```typescript
// Буква T — это просто соглашение (от слова Type). Можно использовать любое имя.
function identity<T>(arg: T): T {
  return arg;
}

// 1. Явное указание типа (Передаем тип в "переменную" T)
const result1 = identity<string>("hello"); // T становится string, result1 имеет тип string

// 2. Автоматический вывод типов (Type Inference)
// TypeScript сам смотрит на аргумент и понимает, какой тип подставить в T
const result2 = identity(42); // TS сам догадался, что T — это number
```

---

## 🏗 2. Дженерики в интерфейсах и типах

Мы можем создавать универсальные структуры данных, форма которых зависит от переданного типа.

```typescript
// Описываем универсальную коробку для чего угодно
interface Box<T> {
  content: T;
}

const stringBox: Box<string> = { content: "Книга" };
const numberBox: Box<number> = { content: 100 };
```

---

## ⛓ 3. Constraints (Ограничения дженериков через `extends`)

По умолчанию в `<T>` можно передать абсолютно любой тип. Но часто нам нужно поставить условие: _«Сюда можно передать не любой тип, а только тот, который обладает определенными свойствами»_. Для этого используется ключевое слово `extends`.

```typescript
interface HasLength {
  length: number;
}

// Ограничиваем T: он ОБЯЗАН быть подтипом HasLength (иметь свойство length)
function logLength<T extends HasLength>(arg: T): void {
  console.log(arg.length); // Теперь это легально! TS уверен, что length существует.
}

logLength("hello");      // Успешно (у строк есть .length)
logLength([1, 2, 3]);    // Успешно (у массивов есть .length)
logLength({ length: 4 });// Успешно (объект со свойством length)
// logLength(42);        // ❌ Ошибка TS! У числа number нет свойства length.
```

## 🔑 4. Ограничение по ключам объекта (`keyof` + `extends`)

С помощью комбинации `extends keyof` можно связать два дженерика между собой. Это классический паттерн для функций, которые достают значения из объектов.

```typescript
// T — сам объект
// K — строго один из ключей этого объекта (Union строк-названий)
function getValue<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]; // TS гарантирует, что возвращаемый тип будет точно соответствовать ключу
}

const user = { id: 1, name: "Alex" };

const name = getValue(user, "name"); // Успешно. Тип name — string.
// const age = getValue(user, "age"); // ❌ Ошибка TS! Аргумент "age" не может быть присвоен типу '"id" | "name"'.
```

## 🪐 5. Дефолтные значения (Default Types)

Как и параметры функций в JavaScript, дженерикам можно задавать тип по умолчанию с помощью знака `=`. Если тип не передан явно и его невозможно вычислить, TS возьмет дефолтный.

```typescript
// По умолчанию API будет возвращать строку, если не указано иное
interface ApiResponse<T = string> {
  data: T;
  status: number;
}

const defaultResponse: ApiResponse = { data: "Просто текст", status: 200 }; // T стал string
const customResponse: ApiResponse<number> = { data: 500, status: 200 };     // T стал number
```

## 🧬 6. Продвинутый трюк: Извлечение типов через `infer`

Ключевое слово `infer` (вывести / угадать) можно использовать только внутри Conditional Types (условных типов с `extends ? :`). Оно позволяет попросить TypeScript автоматически заглянуть внутрь дженерика и вытащить оттуда скрытый тип.

Именно этот трюк мы использовали в типе `PromiseAll`, чтобы заглянуть внутрь `Promise<T>`:

```typescript
// Если T — это Промис, "вытащи" тип того, что лежит внутри него во временную переменную U
type UnpackPromise<T> = T extends Promise<infer U> ? U : T;

type Case1 = UnpackPromise<Promise<string>>; // Итог: string (TS заглянул внутрь и вытащил тип)
type Case2 = UnpackPromise<number>;          // Итог: number (это не промис, сработало условие "иначе")
```

## 💡 Резюме для самопроверки:

- Дженерик — это просто переменная для типа.
- `extends` — это фильтр/ограничение для этой переменной (что туда можно и нельзя передавать).
- `keyof T` — превращает ключи объекта в список строк для проверки.
- `infer` — инструмент-рентген, который вскрывает чужие дженерики и достает их внутренности.
