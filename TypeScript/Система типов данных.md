Типы данных: boolean, number, string, Array, кортежи, Enum, null и undefined, Any, Symbol, Never.

`Any`

Позволяет присвоить переменной любой тип данных и выполнять с ней **любые операции** (вызывать методы, обращаться к свойствам), даже если их не существует.

```ts
// С использованием ANY 
let variableAny: any = "Привет, мир"; 
variableAny.foo(); // TypeScript промолчит. В рантайме будет КРАШ (ошибка)! variableAny.map(); // TypeScript промолчит. В рантайме КРАШ!
```

`Unknown`

Позволяет присвоить переменной любой тип данных, но **запрещает совершать с ней любые действия** (вызывать функции, читать свойства, присваивать другим строгим типам) до тех пор, пока вы не докажете TypeScript, какой именно тип там находится.

```ts
// С использованием UNKNOWN 
let variableUnknown: unknown = "Привет, мир"; variableUnknown.foo(); // ОШИБКА КОМПИЛЯЦИИ: Объект имеет тип 'unknown' 
variableUnknown.map(); // ОШИБКА КОМПИЛЯЦИИ: Объект имеет тип 'unknown'
```

**Безопасность:** Максимальная. Он заставляет разработчика писать проверки кода (сужать тип) перед использованием данных.

`Void`

Этот тип означает отсутствие конкретного типа. Основное предназначение типа `Void` — явно указывать на то, что у функции или метода отсутствует возвращаемое значение.

`Never`

`Never` служит для указания того, что какие-либо операции никогда не будут выполнены.

---

**Описание иерархии типов**

В отличие от JavaScript, где типы существуют только в рантайме (при выполнении кода), в TypeScript типы образуют строгую иерархию от самого всеобъемлющего (Top Type) до абсолютно пустого (Bottom Type).

---

## 🔝 1. Top Types (Верхушка иерархии)

Эти типы могут принимать абсолютно любые значения. В них можно записать всё: от числа до сложного объекта.

- `any` — _«Отключить TypeScript»_. Говорит компилятору полностью игнорировать проверку типов для этой переменной. Позволяет вызывать любые методы, даже если их не существует. Использование в продакшене — дурной тон.
- `unknown` — _«Безопасный `any`»_. Принимает любое значение, но не позволяет ничего с ним делать (вызывать методы, передавать в функции), пока ты явно не сузишь тип с помощью `Type Guard` (например, `typeof` или `instanceof`).

---

## 🧱 2. Primitive Types (Примитивные типы)

Соответствуют базовым примитивам из JavaScript.

- `string` — Любые текстовые строки.
- `number` — Все числа (включая целые, с плавающей точкой, а также `NaN` и `Infinity`).
- `boolean` — Строго `true` или `false`.
- `symbol` — Уникальные идентификаторы.
- `bigint` — Числа произвольной точности (например, `10n`).

---

## 🛍 3. Literal Types (Литеральные типы)

Самый недооцененный инструмент новичками. Это типы, которые состоят из конкретных значений.

```typescript
type Direction = "Left" | "Right"; // Строковый литерал
type Prime = 2 | 3 | 5;             // Числовой литерал
```

- Переменная типа `Direction` может принять _только_ строку `"Left"` или `"Right"`. Любая другая строка вызовет ошибку.
- Template Literal Types (Шаблонные литералы): Позволяют генерировать типы динамически: ``type Event = `on${Direction}`;`` (Итог: `"onLeft" | "onRight"`).

---

## 📦 4. Structural Types (Структурные типы)

TypeScript использует структурную типизацию («утиную типизацию»). Ему важна форма (структура) объекта, а не его имя или класс.

- `object` — Представляет любой непримитивный тип. Всё, что не является числом, строкой, булевом и т.д. (включая массивы и функции).
- `{}` (пустой объект) — Означает «любое значение, кроме `null` и `undefined`». В переменную такого типа можно записать даже число или строку.
- Интерфейсы / Типы (`interface` / `type`) — Описание конкретной структуры объекта:
    
    ```typescript
    type Point = { x: number; readonly y?: number }; // y — опциональный и только для чтения
    ```
    

---

## 🧬 5. Коллекции и Массивы

- `T[]` или `Array<T>` — Массив элементов одинакового типа `T`.
- `readonly T[]` — Неизменяемый массив (у него вырезаны методы `push`, `pop` и т.д.).
- Tuples (Кортежи) — Массив с фиксированной длиной и известными типами на каждой позиции:
    
    ```typescript
    type HttpResponse = [number, string]; // Семплирует [200, "OK"]
    ```
    

---

## 🧊 6. Специальные типы отсутствия значения

- `null` — Намеренное отсутствие объекта.
- `undefined` — Значение не присвоено (дефолтное состояние в JS).
- `void` — Используется исключительно как возвращаемое значение функций. Означает: _«Функция завершается, но ничего полезного не возвращает»_ (под капотом возвращается `undefined`).

---

## 🕳 7. Bottom Type (Самый низ иерархии)

Тип, который находится в самом низу и не может содержать ни одного значения.

- `never` — _«Этого никогда не произойдет»_. Используется в двух случаях:
    
    1. Функция никогда не завершается (например, бесконечный цикл или она всегда выбрасывает `Error`).
    2. При сужении типов, когда все возможные варианты уже были исключены (используется для `exhaustive checks` — проверки на полноту).
    

---

## 🧭 Как работает Иерархия (Ментальный трюк для `extends`)

Когда ты пишешь `A extends B` в условных типах, ты спрашиваешь: «Является ли A подтипом B (находится ли он ниже или на том же уровне)?».

```markdown
      [ any / unknown ]      <-- Верхний уровень
              |
          [ object ]
        /     |    \
   [ Array ] [ {} ] [ Функция ]

      |       |
  [Кортеж] [string]          <-- Примитивы и структуры
              |
        ["hello"]            <-- Литералы
              |
          [ never ]          <-- Нижний уровень
```

## Пример проверки по иерархии:

```typescript
type Check1 = "hello" extends string ? true : false; // true (Литерал находится внутри string)
type Check2 = string extends unknown ? true : false; // true (string находится внутри unknown)
type Check3 = string extends never ? true : false;   // false (string шире, чем пустой never)
```
