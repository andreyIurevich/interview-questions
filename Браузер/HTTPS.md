**HTTP** — передаёт данные в открытом виде
**HTTPS** — передаёт данные **в зашифрованном виде** с проверкой подлинности сервера

HTTPS = HTTP + TLS (ранее SSL)

Как устанавливается HTTPS (TLS handshake):
1. Клиент → сервер: "Хочу подключиться по HTTPS, поддерживаю такие алгоритмы"
2. Сервер → клиент: "Вот мой SSL-сертификат + публичный ключ"
3. Клиент:
	- проверяет сертификат через **Центр сертификации (CA)**
	- убеждается, что домен совпадает
4. Сервер: расшифровывает ключ приватным ключом
Дальше весь трафик идёт по **симметричному шифрованию**

SSL-сертификат содержит:
- домен (`example.com`)
- публичный ключ
- срок действия
- CA, который его выдал
- цифровую подпись

**TLS — это криптографический протокол**, который обеспечивает безопасную передачу данных по сети: шифрование(данные нельзя прочитать при перехвате), проверку подлинности (клиент уверен, что общается с настоящим сервером) и целостность (данные нельзя незаметно изменить).

Где используется TLS:
- HTTPS
- WSS (WebSocket Secure)
- SMTP(S), IMAPS, FTPS
- HTTP/2 и HTTP/3 (обязательно)
