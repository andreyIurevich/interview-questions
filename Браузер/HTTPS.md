**HTTP** — передаёт данные в открытом виде
**HTTPS** — передаёт данные **в зашифрованном виде** с проверкой подлинности сервера

HTTPS = HTTP + TLS (ранее SSL)

Как устанавливается HTTPS (TLS handshake):
1. Клиент → сервер: "Хочу подключиться по HTTPS, поддерживаю такие алгоритмы"
2. Сервер → клиент: "Вот мой SSL-сертификат + публичный ключ"
3. Клиент:
	- проверяет сертификат через **Центр сертификации (CA)**
	- убеждается, что домен совпадает
4. Сервер: расшифровывает ключ приватным ключом
Дальше весь трафик идёт по **симметричному шифрованию**

SSL-сертификат содержит:
- домен (`example.com`)
- публичный ключ
- срок действия
- CA, который его выдал
- цифровую подпись

**TLS — это криптографический протокол**, который обеспечивает безопасную передачу данных по сети: шифрование(данные нельзя прочитать при перехвате), проверку подлинности (клиент уверен, что общается с настоящим сервером) и целостность (данные нельзя незаметно изменить).

**Пошаговый процесс: Как они работают вместе в динамике**

Когда вы вводите адрес сайта (например, `https://example.com`), браузер выполняет строго последовательную цепочку действий:

1. **Создается канал (Уровень TCP)**: Браузер стучится на сервер (обычно на **443 порт** для HTTPS) и проводит стандартное TCP-рукопожатие. Появляется «труба» для передачи байт.
2. **Включается страж (Уровень TLS)**: Прежде чем отправить первый HTTP-запрос, браузер и сервер проводят TLS-рукопожатие внутри этой TCP-трубы. Они проверяют сертификаты и договариваются о секретном ключе шифрования.
3. **Запуск конвейера (Интеграция)**: Теперь, когда TLS-сессия активна, прикладной уровень (HTTP) передает свой запрос (`GET /...`) не напрямую в сеть, а функции шифрования TLS.
4. **Передача данных**: TLS превращает читаемый HTTP-текст в криптографический шум (**TLS Record**), добавляет к нему проверочный код целостности (MAC) и только в таком виде отдает протоколу TCP для отправки по интернету.
5. **Прием на сервере**: Сервер получает зашифрованные байты из TCP, с помощью своего TLS-модуля расшифровывает их и передает обратно наверх — веб-серверу (Nginx, Apache) в виде обычного, чистого HTTP-запроса.

Где используется TLS:
- HTTPS
- WSS (WebSocket Secure)
- SMTP(S), IMAPS, FTPS
- HTTP/2 и HTTP/3 (обязательно)
